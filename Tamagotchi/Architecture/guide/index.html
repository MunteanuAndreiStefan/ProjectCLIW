The problem we start from

This problem was proposed to us during cliw lab, you can find more about it here.

The video presentation can be found here

Create a MV * mini-framework for the development of vector-driven vector-based web applications at the client level. As an actual case study, it will be used to implement a Tamagotchi game simulator. At the very least, each authenticated user will manage one or more characters he will take care of (nourish, educate, amuse). The Web application will provide an interactive way to create and configure these characters, plus sharing them to users. Bonus: Implementing a level message routing system client to build SPA-Single Page Applications.

Introduction

This guide contains instructions in order to use our application.

How to use it

In order to understand our application, we present some printscreens with text associated when needed.

If you are not registered the following options are displayed at the top of the page:Sign In and Sign Up.

If you have an account already you can pick Sign In and enter your credentials - username and password. If you'd like an account, you can pick Sign Up and enter an email address and password in order to make a new one.

If you are authenticated, you can see your tamagotchies in the management list. Each of them has the following options:Edit, View, Share and Delete. You can also make a new tamagotchi.

Add/Edit are similar, but the edit operates on an existing tamagotchi. The options are face shape, eyes, mouth, torso, and face color. In the rectangle under the color selectors you can previsualise the color. Press Apply changes in order to apply the color.

If you press view you can visualise the tamagotchi and play with it by using some buttons.

You have the possibility also to delete the tamagotchi.

If you press share, you will be asked for an user id. He/she will get your tamagotchi. If you'd like someone to send you a tamagotchi, give them your ID (you can see it in the management page).
