The problem we start from

This problem was proposed to us during cliw lab, you can find more about it here.

Create a MV * mini-framework for the development of vector-driven vector-based web applications at the client level. As an actual case study, it will be used to implement a Tamagotchi game simulator. At the very least, each authenticated user will manage one or more characters he will take care of (nourish, educate, amuse). The Web application will provide an interactive way to create and configure these characters, plus sharing them to users. Bonus: Implementing a level message routing system client to build SPA-Single Page Applications.

Structure

Our project Tama should have a html document that is a valid HTML document that follows some additional rules to specialize its meaning and make it predictable to processors wishing to produce or consume scholarly articles. These rules are outlined in the following sections.

Static pages

The static pages will be the ones that give the first impresion to our user. So it's very important to give a good first impresion, we decided to use HTML5, CSS3, jQuery, Modernizr, Accordion and responsiveImg for creating those. There will be 8 pages and 2 txt files in total that will be available as static content for this page, this page will be transformed into a SPA at integration phase:

  • Home - The page in which the user can create his account or login and see updates/promo materials from us.
  • News - This page will be an articles format page on which we will post in order the updates for end user.
  • Development Blog - This page will be an articles format page, it will contain multiple decision we made and sketches.
  • Help - This page will contain a manual for the end user into how to use our application.
  • Contact - This page will contain form. That will have as scope sending a message to us and a iframe with our location.
  • Terms and conditions - This page will contain all our privacy policy and content privacy.
  • Testimonials - This page will contain user feedback about us.
  • 404 error - This page will treat the 404 error.
  • Humans - This text file will contain the team description.
  • Robots - This text file will contain the paths definition for SEO crawlers.

Edit page

This page allows the user to edit the tamagotchi characters. There are many available options so that the user can customize the tamagotchi character in the way he/she likes, just like in any other computer/mobile game, where you can set up your profile. By making use of the numerous possible combinations of shapes, colors, or items, there can be created fun and original tamagotchis. The options refer to the following aspects of the tamagotchi: face shape, eyes shape, mouth, accesory, costume, face color.

  • face shape - possible values are based on geometric figures
  • eyes shape - the setting of this parameter is done just like in the case of the face shape
  • mouth - for a plus of expressivity, the mouth can be adjusted in order to reflect the character's feelings
  • costume - the user can choose the character's clothes, from a set of predefined items
  • accesory - cool accesories like hats or helmets are provided in addition to the costume
  • face color - the user can pick any color by selecting the corresponding HSLA (Hue,Saturation,Lightness,Alpha) values

The interface is user-friendly - it adjusts to the dimensions of the user's screen, by properly scaling the tamagotchi and positioning the other page content. The options are scrollable, to assure that the user doesn't lose any information when he/she uses the site on a small screen. The page controls, like the buttons used to iterate through a list of possible values corresponding to a certain option, are intuitive and don't require the user to read a complicate guide in order to use the application. After the user is done editing, he/she can save the changes by clicking Save.
